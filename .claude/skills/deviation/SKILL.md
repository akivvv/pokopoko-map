# 逸脱(Deviation)の記録

## 逸脱にあたるもの(判断に迷ったら「あたる」扱いで記録する)

- docs/spec/DECISIONS.md と違う実装をした
- 仕様が曖昧で、挙動を自分で仮決めした(テストの期待値の仮決めも含む)
- 計画していた構成・手順を実装の都合で変えた

## 手順

1. `docs/IMPLEMENTATION-NOTES.md` を開き、`## Deviations` セクションの
   直後(最新が上)に以下のテンプレで追記する:

   ```
   ### YYYY-MM-DD: 一行タイトル
   - 計画: 本来どうするはずだったか(DECISIONS §N など参照付き)
   - 実際: どう実装したか
   - 理由: なぜ逸脱したか
   - 影響: 他モジュール・仕様・データへの影響範囲
   - 状態: 暫定(要再検討) / 恒久(DECISIONS反映済み) / 恒久(仕様昇格不要)
   ```

   日付はシステムの今日の日付を使う。「(まだなし)」のプレースホルダが残っていれば消す

2. **状態の付け方**:
   - まず全部「暫定(要再検討)」で登録する
   - 恒久的な仕様変更に相当すると考える場合は、**勝手に DECISIONS.md を書き換えず**、
     ユーザーに昇格を提案する。承認されたら skill: spec-update の手順で反映し、
     このエントリの状態を「恒久(DECISIONS反映済み)」に更新する

3. 完了報告に「逸脱あり(記録済み): タイトル」を必ず含める。逸脱ゼロの場合は何も書かなくてよい

## 禁止事項

- 記録せずに逸脱したまま完了報告する
- 逸脱を隠すためにテストの期待値を実装に合わせて弱める
