# 設計決定の仕様反映と伝播

## 前提

- **確定した決定だけ**を反映する。検討中・提案段階のものは書かない
  (検討メモは `.ai/` に `トピック-YYYY-MM-DD.md` で置く)
- 迷ったら書かずにユーザーに確認する

## 1. DECISIONS.md への反映(docs/spec/DECISIONS.md)

**文体規則(重要)**:
- 「現在どうであるか」だけを現在形で書く。**禁止**: 日付、経緯(「〜と決定した」「〜から変更」)、
  「ユーザー決定」の注記、REBUILD.md の課題番号への言及、選択肢の比較の長文
- 廃止した方式は §9「廃止済みの旧案(混入禁止)」へ1行で追加する
- 先送りは §10「スコープ外・保留」へ

**章の選び方**(既存の章立てを崩さない。新章の追加はユーザー確認):
| 章 | 内容 |
|---|---|
| §1 | アーキテクチャ(構成・ホスティング・Firebase) |
| §2 | 認証・入室(ログイン・招待・認証サーバー・悪用対策) |
| §3 | データモデル(RTDB スキーマ・座標・グリッド) |
| §4 | Security Rules |
| §5 | 同期・キャッシュ(onChild*・LWW・pending・localStorage) |
| §6 | フロントエンド実装方針(ストア・型・ライブラリ・canvas) |
| §7 | UIデザイン「ポコポップ」(トークン) |
| §8 | UI・機能の要点 |
| §9 | 廃止済みの旧案(混入禁止) |
| §10 | スコープ外・保留 |

## 2. 経緯の記録

決定に至る議論・比較・理由は `.ai/DECISION-LOG-2026-07-05.md` に追記するか、
新しいトピックなら `.ai/` に新ファイルを作る(ローカル専用。コミットされない)

## 3. 伝播チェックリスト

決定の影響を受けるファイルを更新する:
- [ ] `.claude/agents/*.md` — 担当エージェントの契約(spec-guardian / map-canvas / firebase-sync /
      auth-server / ui-builder / test-writer / code-reviewer のうち関係するもの)
- [ ] `CLAUDE.md` — 必須ルールに関わる場合のみ
- [ ] 章番号を変えた場合は全参照を追随させ、最後に必ず整合確認:
      ```
      grep -rn "DECISIONS" .claude/agents/ CLAUDE.md docs/
      ```
      参照している章番号が実在し内容が一致するか目視確認する

## 4. 締め

- **コミットしない**(次のまとまったコミットに同梱する。skill: ship 参照)
- 反映した章・伝播先ファイルの一覧を報告する
